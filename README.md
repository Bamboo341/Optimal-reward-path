# OSM報酬最大化経路探索システム

地図（OpenStreetMap）上の道路に「報酬（ポイント）」を自由に設定し、
出発点・到着点・歩ける距離の上限を指定すると、**上限内でできるだけ多くの
報酬を集められる経路**を探して地図に表示するツールです。
すべて自分のパソコンの中で動きます（仕様の詳細は [docs/SPEC.md](docs/SPEC.md)）。

---

## 起動までの全体像

| 手順 | やること | 頻度 |
|---|---|---|
| 手順1 | Python をインストールする | 最初の1回だけ |
| 手順2 | このプロジェクトのフォルダを用意する | 最初の1回だけ |
| 手順3 | 黒い画面（ターミナル）を開いてフォルダに移動する | 起動のたび |
| 手順4 | 準備コマンドを実行する | 最初の1回だけ |
| 手順5 | アプリを起動する | 起動のたび |

初回は手順1〜5をすべて、**2回目以降は手順3と5だけ**でOKです
（後述の「2回目以降の起動方法」参照）。

## 手順1: Python をインストールする（最初の1回だけ）

このツールは Python というソフトの上で動きます。**バージョン 3.11 以上**が必要です。

### Windows の場合

1. ブラウザで https://www.python.org/downloads/ を開く
2. 黄色い「Download Python 3.x.x」ボタンを押してインストーラーをダウンロード
3. ダウンロードしたファイルをダブルクリックして実行
4. **重要**: 最初の画面の下にある **「Add python.exe to PATH」に必ずチェック**を入れる
   （これを忘れると後のコマンドが動きません）
5. 「Install Now」を押してインストール完了まで待つ

**確認方法**: スタートメニューで「cmd」と入力して「コマンドプロンプト」を開き、

```
python --version
```

と入力して Enter。`Python 3.11.5` のように 3.11 以上の数字が表示されれば成功です。
「'python' は、内部コマンド…として認識されていません」と出た場合は、
手順4のチェックを入れ忘れています。インストーラーをもう一度実行し、
「Uninstall」→再インストールでチェックを入れ直してください。

### Mac の場合

1. 「ターミナル」アプリを開く（Launchpad で「ターミナル」と検索）
2. `python3 --version` と入力して Enter
3. 3.11 以上なら手順2へ。それより古い・入っていない場合は
   https://www.python.org/downloads/ からインストールしてください

## 手順2: プロジェクトのフォルダを用意する（最初の1回だけ）

このリポジトリの内容一式をパソコンに置きます。どちらかの方法で:

- **ZIPでダウンロードする場合**: GitHub のリポジトリページで緑の「Code」ボタン →
  「Download ZIP」→ ダウンロードした ZIP を右クリックして「すべて展開」。
  展開先はどこでも構いません（例: `C:\Users\あなたの名前\Optimal-reward-path`）
- **Git を使える場合**: `git clone <リポジトリのURL>`

以降の説明では、このフォルダを「プロジェクトフォルダ」と呼びます。
フォルダの中に `app.py` や `requirements.txt` というファイルがあることを確認してください。

## 手順3: 黒い画面（ターミナル）を開いてフォルダに移動する

コマンドを入力するための画面を開き、作業場所をプロジェクトフォルダにします。

### Windows の場合（いちばん簡単な方法）

1. エクスプローラーで**プロジェクトフォルダを開く**（`app.py` が見えている状態）
2. ウィンドウ上部の**アドレスバーをクリック**し、`cmd` と入力して Enter
3. 黒い画面が開きます。このとき最初から目的のフォルダにいる状態になっています

### Mac の場合

1. ターミナルを開く
2. `cd ` （cdの後に半角スペース）と入力し、プロジェクトフォルダを
   ターミナルの画面に**ドラッグ＆ドロップ**して Enter

**確認方法**: `dir`（Windows）または `ls`（Mac）と入力して Enter。
一覧に `app.py` が表示されていれば正しい場所にいます。

## 手順4: 準備コマンドを実行する（最初の1回だけ）

手順3で開いた黒い画面に、次のコマンドを**1行ずつ**入力して Enter を押します。

### 4-1. 仮想環境を作る

このツール専用の部屋（仮想環境）を作ります。パソコンの他の部分を汚しません。

```
python -m venv .venv
```

※ Mac では `python` の代わりに `python3` と入力してください。
何も表示されずに次の行に戻れば成功です（数十秒かかることがあります）。

### 4-2. 仮想環境に入る（有効化）

- **Windows（コマンドプロンプト）**:

  ```
  .venv\Scripts\activate
  ```

- **Windows（PowerShell を使っている場合）**:

  ```
  .venv\Scripts\Activate.ps1
  ```

  ※「スクリプトの実行が無効」というエラーが出たら、上のコマンドプロンプト版を
  使うのが簡単です（手順3で `cmd` と入力して開き直す）。

- **Mac**:

  ```
  source .venv/bin/activate
  ```

**確認方法**: 行の先頭に `(.venv)` と表示されるようになれば成功です。

### 4-3. 必要な部品をまとめてインストールする

```
pip install -r requirements.txt
```

地図処理などに必要なライブラリを自動でダウンロードします。
**5〜10分かかることがあります**。インターネット接続が必要です。
最後にエラーなくプロンプト（入力待ち）に戻れば成功です。

## 手順5: アプリを起動する

```
streamlit run app.py
```

- 初回だけ `Email:` と聞かれることがあります → **何も入力せず Enter** で進めます
- 自動でブラウザが開き、アプリの画面が表示されます。
  開かない場合は、画面に表示される `Local URL: http://localhost:8501` を
  ブラウザのアドレスバーに入力してください
- **初回起動時のみ**、地図データ（既定: 京都市下京区）をインターネットから
  取得するため**数分待ちます**（「道路ネットワークを読み込み中...」と表示されます）。
  取得したデータは `data/` フォルダに保存され、**2回目以降は数秒で起動**します

### アプリの終了方法

黒い画面をクリックして **Ctrl キーを押しながら C** を押します。
（ブラウザのタブを閉じるだけではアプリは止まりません）

## 使い方（画面の操作）

画面左のサイドバーで「報酬設定」と「経路探索」を切り替えます。

### 1. 報酬設定 — 道路にポイントを付ける

1. 「選択可能な道路をグレーで表示」にチェックが入っていると、クリックで
   選択できる道路がグレーの線で表示される（狙いやすくなります）
2. 地図上の、報酬を付けたい道路の近くをクリックする
   （クリックした位置に黒い丸が表示されます）
3. いちばん近い道路が**青色**で表示され、サイドバーに道路名が出る
4. サイドバーで報酬値（1以上の整数）とメモ（任意）を入力し「設定」を押す
5. その道路が**赤色**に変わり、設定が即座にファイルへ保存される
   （アプリを再起動しても消えません）
6. 画面下部の一覧から、報酬値の変更（「更新」）や「削除」ができる

### 2. 経路探索 — 報酬を最大化する経路を探す

1. サイドバーで「出発点」を選び、地図上の出発したい場所をクリック
   （緑のマーカーが立ちます）
2. サイドバーで「到着点」に切り替え、到着したい場所をクリック
   （赤のマーカーが立ちます）
3. 距離上限 L（歩ける距離の上限、メートル）を入力する
4. モードを選ぶ:
   - **再訪許可（歩道）**: 同じ道を何度通ってもよい
   - **再訪禁止（小道）**: 同じ道路は一度しか通らない
5. 候補数 K（探索する経路の数、1〜5）を選び「探索実行」を押す
6. 地図の上の「地図に表示する候補」で選んだ候補だけが地図に描画される
   （既定は候補1。「すべて表示」で全候補の重ね表示もできる）。
   表には各候補の総報酬・総距離・距離上限の使用率・回収した報酬エッジ数が出る
7. 「GeoJSONエクスポート」ボタンで、候補経路を `output/` フォルダに
   ファイル保存できる（GIS ソフト等で開ける形式です）

※ 出発点から到着点までの最短距離が距離上限より長い場合は、
「実行可能解がありません」とエラーが表示されます。距離上限を増やしてください。

## 設定を変えたいとき（config.yaml）

プロジェクトフォルダの `config.yaml` をメモ帳などで編集します。

| キー | 既定値 | 説明 |
|---|---|---|
| `place` | `"Shimogyo-ku, Kyoto, Japan"` | 対象地域。例: `"Kyoto, Japan"` に変えると京都市全域 |
| `network_type` | `"all"` | 対象とする道路の種類 |
| `n_exact` | `14` | 回収可能な報酬エッジ数がこの値以下なら厳密解、超えると高速な近似解 |
| `time_limit_sec` | `90` | 近似探索の時間上限（秒） |
| `k_default` | `3` | 候補数 K の初期値 |
| `data_dir` / `output_dir` | `data` / `output` | データ保存先フォルダ |

`place` を変えた場合、その地域の地図データを初回に取得します（要インターネット）。
報酬データは地域ごとに別ファイルで保存されます。

## よくあるトラブルと対処

| 症状 | 対処 |
|---|---|
| `'python' は認識されていません` | 手順1の「Add python.exe to PATH」のチェック漏れ。Python を入れ直す |
| PowerShell で `Activate.ps1` がエラー | コマンドプロンプト（手順3で `cmd`）を使う |
| `pip install` が失敗する | インターネット接続を確認。`python -m pip install --upgrade pip` を実行してから再度試す |
| 初回のグラフ取得に失敗する | 接続を確認（社内プロキシ環境では openstreetmap.org への接続許可が必要）。別環境で取得した `data/graph_～.graphml` をコピーして置いても動きます |
| `Port 8501 is already in use` | すでに起動中です。既存の黒い画面を Ctrl+C で止めるか、`streamlit run app.py --server.port 8502` で別ポートを使う |
| 「報酬ファイル…壊れています」と表示される | `data/rewards_～.json` を手で編集して壊れた状態です。修正するか削除してください（削除すると報酬設定は消えます） |
| 探索が遅い | 報酬エッジが多い場合は `config.yaml` の `time_limit_sec` を小さくすると早く打ち切ります（その時点の最良解を返します） |

## 開発者向け情報

### テストの実行

```
pytest tests/
```

合成グラフによる既知最適解の検証・ブルートフォースとの照合・報酬の1回加算・
再訪禁止モードのエッジ重複なし検証などを含みます（ネットワーク接続は不要）。

### プロジェクト構成

```
├── app.py               # Streamlit エントリポイント（2画面の切替）
├── config.yaml          # 設定
├── src/
│   ├── graph_loader.py  # OSM取得・graphmlキャッシュ・無向化・最近傍探索
│   ├── rewards.py       # 報酬データのCRUD・JSON永続化
│   ├── route.py         # 経路復元・報酬再集計・実行可能性検証
│   ├── export.py        # GeoJSONエクスポート
│   ├── solver/
│   │   ├── metagraph.py # メタグラフ構築・最短距離前計算
│   │   ├── exact_dp.py  # ビットマスクDP（厳密解）
│   │   ├── heuristic.py # 貪欲＋ローカルサーチ（|R|大）
│   │   └── mode_s.py    # 再訪禁止モードの経路構築
│   └── ui/              # Streamlit 画面部品（報酬設定・経路探索・地図）
├── tests/               # pytest（合成グラフ・ネットワーク非依存）
├── data/                # グラフキャッシュ・報酬データ（git管理外）
└── output/              # GeoJSONエクスポート先（git管理外）
```
