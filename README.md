# OSM報酬最大化経路探索システム

OpenStreetMap の道路ネットワーク上で道路エッジに報酬（ポイント）を手動設定し、
出発点・到着点・距離上限を指定して**取得報酬を最大化する経路**を探索・表示する
ローカルツールです。仕様の詳細は [docs/SPEC.md](docs/SPEC.md) を参照してください。

## 必要環境

- Python 3.11 以上
- 初回のグラフ取得時のみインターネット接続（OSM / Nominatim / Overpass へのアクセス）

## セットアップ

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## 起動

```bash
streamlit run app.py
```

ブラウザが自動で開きます（開かない場合は表示されるURL、通常 http://localhost:8501 を開く）。

- 初回起動時は OSM から対象地域（既定: 京都市下京区）の道路ネットワークを取得する
  ため数分かかることがあります。取得したグラフは `data/graph_{place_slug}.graphml`
  にキャッシュされ、2回目以降は数秒で読み込まれます
- 取得に失敗した場合はエラーが表示されます。ネットワーク接続を確認するか、
  取得済みの graphml キャッシュを `data/` に配置してください

## 使い方

### 1. 報酬設定（サイドバー「報酬設定」）

1. 地図上の任意の点をクリック → 最近傍の道路エッジが青色でプレビューされ、道路名が表示される
2. サイドバーで報酬値（正整数）とメモを入力し「設定」→ 即時に JSON へ保存される
3. 設定済みエッジは赤色＋報酬値ツールチップで表示。画面下部の一覧から報酬値の変更・削除が可能

報酬データは `data/rewards_{place_slug}.json` に保存され、再起動後も保持されます。

### 2. 経路探索（サイドバー「経路探索」）

1. 「出発点」「到着点」を選んで地図をクリック → 最近傍ノードにスナップしマーカー表示
2. 距離上限 L（メートル）、モード、候補数 K（1〜5）を指定して「探索実行」
   - **再訪許可（歩道）**: 同一エッジ・ノードの再訪自由
   - **再訪禁止（小道）**: 同一の道路エッジを2度通らない経路のみ
3. 候補経路が色分けして地図に重畳表示される（レイヤーコントロールで表示/非表示切替）。
   候補ごとの総報酬・総距離・距離上限使用率・回収エッジ数が表で表示される
4. 「GeoJSONエクスポート」ボタンで候補経路を `output/` に保存できる
   （LineString、properties: reward / length / collected_edges）

出発点から到着点への最短距離が距離上限を超える場合はエラーが表示されます。

## 設定（config.yaml）

| キー | 既定値 | 説明 |
|---|---|---|
| `place` | `"Shimogyo-ku, Kyoto, Japan"` | 対象地域。変更するだけで京都市全域等に切替可能 |
| `network_type` | `"all"` | OSMnx の道路種別 |
| `n_exact` | `14` | 回収可能な報酬エッジ数がこの値以下なら厳密解（ビットマスクDP）、超えるとヒューリスティック |
| `time_limit_sec` | `90` | ヒューリスティック探索の時間上限（秒） |
| `k_default` | `3` | 候補数 K の既定値 |
| `data_dir` / `output_dir` | `data` / `output` | データ保存先 |

`place` を変更した場合、初回はその地域のグラフ取得（ネットワーク接続）が必要です。
報酬データは地域ごとに別ファイルで管理されます。

## テスト

```bash
pytest tests/
```

合成グラフによる既知最適解の検証・ブルートフォースとの照合・報酬の1回加算・
モードSのエッジ重複なし検証などを含みます（ネットワーク接続は不要）。

## プロジェクト構成

```
├── app.py               # Streamlit エントリポイント（2画面の切替）
├── config.yaml          # 設定
├── src/
│   ├── graph_loader.py  # OSM取得・graphmlキャッシュ・無向化・最近傍探索
│   ├── rewards.py       # 報酬データのCRUD・JSON永続化
│   ├── route.py         # 経路復元・報酬再集計・実行可能性検証
│   ├── export.py        # GeoJSONエクスポート
│   ├── solver/
│   │   ├── metagraph.py # メタグラフ構築・最短距離前計算
│   │   ├── exact_dp.py  # ビットマスクDP（厳密解）
│   │   ├── heuristic.py # 貪欲＋ローカルサーチ（|R|大）
│   │   └── mode_s.py    # 再訪禁止モードの経路構築
│   └── ui/              # Streamlit 画面部品（報酬設定・経路探索・地図）
├── tests/               # pytest（合成グラフ・ネットワーク非依存）
├── data/                # グラフキャッシュ・報酬データ（git管理外）
└── output/              # GeoJSONエクスポート先（git管理外）
```

## トラブルシューティング

- **初回のグラフ取得に失敗する**: プロキシ・ファイアウォール環境では
  nominatim.openstreetmap.org / overpass-api.de への接続が必要です。
  別環境で取得した `data/graph_{place_slug}.graphml` を配置しても動作します
- **報酬ファイルの破損エラーが出る**: `data/rewards_{place_slug}.json` を修正
  または削除してから再読み込みしてください
- **探索が遅い**: 報酬エッジ数が多い場合は `config.yaml` の `time_limit_sec` を
  調整してください（探索は上限到達時点の最良解を返します）
